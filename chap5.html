Chapter 5: Systeme de Gestion de Version

1) Introduction :

Qu’est-ce que Git ? Git est un outil qui va vous autoriser de faire le versioning, il permet d’avoir l’historique de toutes les modifications que vous faite à un projet. Il permet de résoudre les problèmes qui l’on peut avoir au niveau des conflits.

2) A quoi sert concrètement un système de gestion de version ?

Les logiciels de versioning vont nous permettre d’effectuer ces opérations d’une manière robuste et avec des fonctionnalités utiles. Git permet à différentes versions d’un même fichier de coexister et de coordonner le travail entre plusieurs personnes.

3) Les deux modèles des logiciels de gestion de version :

Les logiciels de gestion de version sont construits sur l’un des deux modèles suivant :

Modèle centralisé

principe de base d’un modèle centralisé est la centralisation du code source lié au projet, utilisateurs doivent se connecter au serveur pour travailler sur ce code.

Modèle décentralisé

Ce modèle invite l’utilisateur de télécharger et à héberger l’intégralité du code source du projet sur sa propre machine. Et parmi ces avantages notables : la simplicité du travail et la sécurité.

4) Qu’est-ce que GitHub ?

GitHub est considéré comme un outil essentiel pour les ingénieurs logiciels, et sa popularité est inégalée. Il est un service en ligne d’hébergement de dépôts Git qui fait office de serveur central pour ces dépôts.

5) Utiliser Git : ligne de commande, console et interface graphique

Pour installer Git nous allons utiliser la console de commande

  • Raccourcie : Windows + R.
  • Dans la barre de recherche, rechercher cmd.

    • 6) Installation de Git :

  • Télécharger la dernière version sur le site http://git-scm.com/downloads.
  • Ouvrir le fichier téléchargé suivre les instructions à l’écran en laissant les valeurs par défaut.

    • 7) Paramétrage de Git :

  • Le Git doit être installé.
  • Utiliser la commande git configpour voir et modifier les variables de configuration qui contrôlent tous les aspects de apparence de Git.
  • Passer option --global à la commande git config pour utiliser les informations globalement.
  • Renseigner votre nom et votre adresse e-mail avec les commandes git config –global user.name « Votre nom » et git config --global user.email « Votre adresse e-mail ».
  • Vérifier que vos informations ont été enregistrées en tapant git config user.name et git config user.email.

    • 8) Démarer un dépôt Git :

    Il existe deux façons de créer un dépôt Git :

  • On peut importer un répertoire déjà existant dans Git.
  • On peut cloner un dépôt Git déjà existant.

    • 9) La gestion des informations selon Git :

    Git gère les informations sous forme d’instantanés ou « snapshots », ce qui permet de réaliser la plupart des opérations localement sans avoir besoin d’une connexion à un serveur central distant, rendant ainsi le travail plus rapide et plus agréable.

    10) Les états des fichiers :

    Git est un outil populaire utilisé pour le contrôle de version dans le développement de logiciels. Il permet de suivre les modifications apportées aux fichiers d’un projet, qui peuvent être soit suivis par une version, soit non suivis. Un fichier est considéré comme « suivi » s’il a été enregistré dans la base de données lors du dernier instantané capturé par Git. En revanche, tout fichier ne faisant pas partie du dernier instantané et n’ayant pas été indexé est « non suivi ». Initialement, tous les fichiers d’un dépôt nouvellement créé sont non suivis et doivent être indexés pour être enregistrés dans la base de données. Cependant, lors du clonage d’un référentiel existant, tous les fichiers sont déjà suivis. Les fichiers suivis peuvent avoir trois états : modifié, indexé (en phase) et validé. Lors du démarrage d’un dépôt Git à partir d’un dépôt local, tous les fichiers doivent être validés, et valider un fichier signifie le stocker dans la base de données locale. Les fichiers nouveaux ou modifiés sont considérés comme « non suivis » ou « modifiés ». Un fichier modifié est considéré comme tel tant qu’il n’a pas été indexé. Indexer un fichier signifie indiquer à Git qu’il doit faire partie du prochain instantané dans sa version actuelle. Enfin, lors de la prise d’un instantané de l’état actuel du projet (c’est-à-dire tous les fichiers indexés et non modifiés), tous les fichiers qui font partie de l’instantané sont considérés comme « validés» à nouveau, et le cycle recommence.

    11) Les zones de travails :

    Git est un système de contrôle de version qui organise les fichiers en états et les relie à des zones de travail. Chaque projet Git dispose d’un arbre de travail, d’une zone de transit et d’un dépôt. L’arbre de travail est une vérification de la version du projet, où les fichiers sont extraits de la base de données compressée et placés sur le disque pour être modifiés. La zone de transit stocke des informations sur les fichiers qui feront partie de la prochaine livraison.Le dépôt est l’endroit où Git stocke les métadonnées et la base de données d’objets du projet. Le processus de travail consiste à modifier ou à créer des fichiers dans le répertoire de travail, à les indexer pour les déplacer vers la zone de stockage et à les valider pour les ajouter au répertoire Git.

    12) Créer un dépôt Git à partir d’un répertoire existant :

  • D’abord on doit créer un document (répertoire) qui contient les fichiers (ressources) qu’on veut utiliser. Note : La commande cd aide à nous positionner à un emplacement dans le pc et dir pour afficher les fichier et dossiers dans l’emplacement actuel. Puis on utilise la commande git init pour initialiser le dépôt Git. On reçoit un message qui nous informe qu’il est bien initialisé. La commande git status est une commande très utile pour déterminer l’état des fichiers. Ensuite on doit indexer nos fichiers et pour cela on utilise la commande git add. Cette étape sert à ce que nos fichiers puissent être validés.
  • Note : les fileglobs utilisent les extensions de chemin de fichier. Suivant pour valider les fichiers et les ajouter en base on utilise la commande git commit .Ça aide à la compréhension des modifications par différents contributeurs en ajoutant un message après la validation.
  • Finalement, on tape: wq entrée ou x et entrée ZZ pour quitter et valider.
  • A la fin tous les fichiers doivent être suivis et enregistrés en base et qu’aucune modification a été apportée depuis le dernier commit.

    • 13) Cloner un dépôt Git:

    Il y a une autre méthode pour démarrer un dépôt c'est de cloner localement un dépôt existant en utilisant la commande Git clone puis créez un repo sur GitHub. Après la création du repo maintenant nous pouvons transmettre nos fichiers à ce repo comme suit :

    git branch -M main.

    git remote add origin http://github.com/Owner_name/ExempleClone.git.

    git push -u origin main.

    Reste à vérifier notre repos for assurer que les fichiers ont été transmis.

    14) Ajouter ou modifier des fichiers dans un projet et actualiser notre dépôt Git

    Notre projet contient un ensemble des fichiers, ainsi qu’un dépôt Git qui sert à gérer les différentes versions de ce projet. Pendant que nous travaillons sur le projet, nous pourrons ajouter, modifier ou supprimer certains fichiers, et pour pouvoir y accéder plus tard (au fichiers précédents), nous devrons effectuer ces modifications dans Git.

    Git add et git commit des commandes qui nous permettre à enregistrer un fichier dans le dépôt Git.

    Le commit va enregistrer notre fichier au moment du Git add.

    Donc pour valider le fichier il faut d’abord effectuer un git add puis un git commit cela va enregistrer les dernières modifications dans le dépôt Git.

    15) Consulter l’historique des modifications Git :

    Git log c’est une commande qui nous permet de consulter l’historique de Git.

    16) Annuler des modifications apportées à un fichier :

    Git checkout --nom du fichier-- ou git restore, sont deux commandes qui nous permet à revenir à l’état enregistrer précédent.

    17) Qu’est-ce qu’une branche ?

    Une branche est une copie de notre projet, dont on peut l’affecter des modifications sans endommager le projet principal.

    La création de cette branche nous emmènera a créé une copie physique, ce qui rendra l’opération plus lourd.

    Une branche dans Git, est simplement un pointeur vers un commit.

    Un pointeur sert à accéder à une donnée.

    Master est une branche sert à pointer sur le dernier commit effectuer.
